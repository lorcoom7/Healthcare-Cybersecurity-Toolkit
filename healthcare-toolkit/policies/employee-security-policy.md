# Employee Cybersecurity Policy

## Healthcare Organization: _____________________________

---

## 1. Purpose

This Employee Cybersecurity Policy establishes requirements for all workforce members to protect Protected Health Information (PHI) and organizational information systems in compliance with HIPAA Security Rule §164.308.

## 2. Scope

This policy applies to:
- All employees
- Contractors
- Temporary staff
- Volunteers
- Students
- Any other workforce members

---

## 3. General Security Responsibilities

| Responsibility | Requirement |
|----------------|-------------|
| Protect credentials | Never share passwords |
| Report incidents | Immediately notify IT |
| Follow policies | Read and acknowledge all policies |
| Complete training | Annual security awareness training |
| Safeguard PHI | Minimum necessary access only |

---

## 4. Acceptable Use

### 4.1 Approved Activities

- [ ] Work-related tasks on organizational systems
- [ ] Limited personal use (email, browsing) during breaks
- [ ] Accessing work email remotely via approved methods
- [ ] Using approved mobile applications

### 4.2 Prohibited Activities

- [ ] Sharing credentials or access tokens
- [ ] Installing unauthorized software
- [ ] Accessing personal email on work devices
- [ ] Using file-sharing services (Dropbox, Google Drive)
- [ ] Visiting illegal or inappropriate websites
- [ ] Connecting personal devices to work network
- [ ] Discussing patient information in public
- [ ] Taking photos of screens or documents
- [ ] Using work devices for personal business
- [ ] Removing data from premises without approval

## 6. Email & Communication Security

### 6.1 Email Best Practices

| # | Practice | Followed |
|---|----------|----------|
| 1 | Verify sender before clicking links | ☐ |
| 2 | Never send PHI via unencrypted email | ☐ |
| 3 | Use secure file sharing for large files | ☐ |
| 4 | Verify requests for sensitive info | ☐ |
| 5 | Don't forward work email to personal | ☐ |
| 6 | Report suspicious emails to IT | ☐ |
| 7 | Use encryption for sensitive data | ☐ |

### 6.2 Phishing Recognition

| Red Flag | Action |
|----------|--------|
| Urgent requests for credentials | Verify via phone |
| Unexpected attachments | Don't open - report to IT |
| Mismatched URLs | Don't click - report to IT |
| Requests for PHI via email | Verify with sender directly |
| Unusual sender addresses | Contact sender to verify |

## 9. Social Engineering Awareness

### 9.1 Common Attack Vectors

| Attack Type | Warning Signs |
|-------------|---------------|
| Phone pretexting | Unsolicited caller asking for info |
| Baiting | Free USB drives or files |
| Tailgating | Following authorized person into secure area |
| Quid pro quo | Services in exchange for information |
| Shoulder surfing | Someone watching your screen |

### 9.2 Defense Checklist

| # | Practice | Followed |
|---|----------|----------|
| 1 | Verify identity of requestors | ☐ |
| 2 | Don't provide info to unsolicited requests | ☐ |
| 3 | Report suspicious contacts | ☐ |
| 4 | Question urgency in requests | ☐ |
| 5 | Don't let strangers follow you in | ☐ |
